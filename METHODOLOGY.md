# 评估方法与证据规则

最后核查：2026-07-17

本项目的目标不是给指纹浏览器排一个看似精确的总榜，而是把“厂商公开了什么、哪些信息可以交叉确认、哪些仍有冲突”整理成可复核的记录。

## 1. 产品纳入标准

首版纳入 AdsPower、Multilogin、GoLogin、Dolphin Anty、MoreLogin 和 Web4 Browser。平台需要同时满足：存在可访问的官方站点或帮助中心；公开资料足以填写至少一组核心字段；与中文用户常见的指纹浏览器、反检测浏览器或多环境浏览器选择有关。

平台名称出现在本项目中，不代表推荐或背书。以后新增平台时，仍需使用相同字段和证据规则。

## 2. 来源优先级

从高到低依次使用：

1. 官方价格页、帮助中心、开发文档、系统要求、隐私政策与版本记录；
2. 同一厂商的其他公开页面，用于解释或交叉确认；
3. 标准组织、浏览器厂商及权威技术文档，用于解释通用概念；
4. 第三方公开资料，仅作为待核查线索，不直接写成确定的产品事实。

搜索结果摘要、广告文案、联盟评测、论坛评论和无法追溯的截图，不单独构成确定结论。

## 3. 证据标签

| 中文标签 | CSV 值 | 使用条件 |
|---|---|---|
| 官方资料 | `official` | 一个公开的官方页面直接支持该记录 |
| 公开核查 | `cross_checked` | 至少两个公开来源相互支持，或官方不同页面能够交叉确认 |
| 编辑分析 | `editorial_analysis` | 基于已列明证据作出的解释，不是厂商原话或实测结果 |
| 实际测试 | `tested` | 同时公开环境、版本、日期、步骤、截图或日志，且能追溯到原始证据 |
| 待核实 | `unverified` | 来源缺失、页面冲突、动态内容无法稳定读取，或暂时无法确认 |

首版没有真实安装和横向测试，因此数据中不使用 `tested`。

## 4. 核心字段如何核查

- `supported_os` 只记录客户端明确支持的系统，不把“浏览器环境可模拟的系统”误写为客户端支持系统。
- `browser_engine` 优先使用产品帮助中心或技术文档，不根据界面外观猜测。
- `automation_interfaces` 只有在官方 API 或帮助文档明确列出时才填写。
- `team_features` 记录角色、权限、环境分享或席位，不把“可分享链接”自动等同于完整团队协作。
- `proxy_support` 区分可接入自定义代理与自带代理流量；本表不评价代理质量。
- `data_storage` 记录厂商公开的本地、云端或混合模式，不推断实际加密强度或数据所在地。
- `free_plan` 记录长期免费额度；限时试用单独写在备注，不与免费版混为一谈。

## 5. 价格标准化

- 保留官网显示的币种、计费周期、环境数和成员数。
- 年付总价与“年付月均价”分开记录，不自行换算。
- 限时折扣、优惠码和新用户价格只有在条件明确时才记录。
- 动态选择器没有稳定显示金额时留空，不从第三方文章补数。
- 价格会随地区、税费、账户注册时间和促销变化；购买前应以结算页为准。

`data/pricing.csv` 不是所有套餐组合的完整价目表，而是保留可稳定引用的免费档、入门档和代表性容量。完整矩阵请回到 `SOURCES.md` 中的官方价格页。

## 6. 来源冲突如何处理

当官方页面互相冲突时：

1. 不静默选择对某个产品更有利的数字；
2. 优先记录日期更近、用途更直接的页面，但保留冲突说明；
3. 无法确定时使用 `unverified`；
4. 在下一次核查时重新访问相关页面。

例如，本次核查发现 GoLogin 价格帮助页的计划表写年付月均 4.5 美元起，同页 FAQ 示例却重复写成 9 美元。该年付记录因此标为 `unverified`，购买前需以结算页确认。

## 7. 事实与编辑分析分离

Markdown 对比中的“更适合先核对什么”属于编辑分析，只能根据已列明的字段推导。以下表达在没有实测前不会使用：

- 成功率最高、最安全、最稳定；
- 一定不会关联或封号；
- 风控通过率、指纹通过率、验证码成功率；
- 最佳、第一名、闭眼选；
- 已长期实测、独立第三方认证。

## 8. 未来实际测试的最低要求

只有同时公开以下内容，记录才可以使用 `tested`：

- 客户端和浏览器内核版本；
- 操作系统、硬件与网络条件；
- 代理类型及控制变量；
- 测试日期、步骤、样本量与失败定义；
- 原始截图、日志或结构化结果；
- 已知偏差、复测条件和不可外推范围。

厂商演示、编辑体验、单次成功或没有原始证据的“独立测试”不满足该标准。

## 9. 为什么不做综合评分

价格、自动化、团队权限、本地存储和跨平台支持的价值取决于使用场景。把这些维度压成一个 100 分总分，会隐藏权重选择并制造虚假精确。项目先给出事实字段、证据等级和限制，让读者按自己的约束作选择。

## 10. 更新规则

高频变化的价格与版本优先复核；来源失效或字段变化写入 `CHANGELOG.md`。任何人都可以提交 issue 指出来源冲突，但修改事实时必须给出可访问的公开链接和核查日期。
